## 📦 安装步骤

### 1. 准备图标文件
在 `icons/` 目录下确保有以下PNG图标：
- `icon16.png` (16×16像素)
- `icon48.png` (48×48像素)  
- `icon128.png` (128×128像素)

*详细图标说明请查看 `icons/README.md`*

### 2. 加载到Chrome浏览器

1. 打开Chrome浏览器
2. 访问扩展管理页面：
   - 地址栏输入 `chrome://extensions/`
   - 或通过菜单：更多工具 → 扩展程序
3. 开启右上角的**开发者模式**开关
4. 点击**加载已解压的扩展程序**
5. 选择本项目的根目录文件夹
6. 安装成功！在工具栏可看到扩展图标

### 3. 开始使用

点击浏览器工具栏中的🔐图标，即可打开加密工具箱界面。

## 🔧 详细使用说明

### AES对称加密使用

1. **密钥准备**：
   - 点击"生成"按钮创建安全密钥（默认256位）
   - 或选择128位密钥长度后生成
   - 密钥采用Base64格式，安全易用

2. **加密操作**：
   - 在明文框输入要加密的内容
   - IV留空将自动生成（推荐）
   - 点击"🔒 加密"或按Ctrl+Enter
   - 结果使用`IV.密文`格式，便于传输

3. **解密操作**：
   - 输入密钥和密文（支持组合格式）
   - 点击"🔓 解密"进行解密
   - 支持新旧格式自动识别

### RSA非对称加密使用

1. **密钥生成**：
   - 选择密钥长度（2048/3072/4096位）
   - 点击"生成密钥对"
   - 系统生成加密和签名两套密钥（兼容模式）

2. **加密流程**：
   - 输入明文（注意长度限制）
   - 使用公钥进行加密
   - 获得Base64格式密文

3. **解密流程**：
   - 输入密文和对应私钥
   - 执行解密获得原文

### RSA数字签名使用

1. **选择签名算法**：
   - **RSA-PSS（推荐）**：更安全的现代标准
   - **SHA256withRSA**：传统兼容格式

2. **数字签名**：
   - 选择合适的算法
   - 输入要签名的消息
   - 提供RSA私钥
   - 点击"✍️ 数字签名"生成签名

3. **签名验证**：
   - 输入原始消息和签名
   - 提供对应的RSA公钥
   - 选择相同的算法
   - 点击"✅ 验证签名"
   - 查看彩色验证结果

### 哈希算法使用

1. **选择算法**：SHA-1/SHA-256（推荐）/SHA-384/SHA-512
2. **输入文本**：在文本框中输入要哈希的内容
3. **计算哈希**：点击"🔗 计算哈希"
4. **查看结果**：显示算法、位长度和哈希值

### 安全工具使用

1. **密码生成**：
   - 设置长度（8-64字符）
   - 选择字符类型（包含特殊字符）
   - 点击"🎲 生成密码"

2. **强度检查**：
   - 输入密码即可实时查看强度
   - 获得评分和改进建议

3. **数据清理**：
   - "清理所有字段"：清除所有敏感信息
   - "仅清理密钥"：只清理密钥相关数据

## 🐛 故障排除

### 常见问题解答

**Q: 扩展无法加载？**
A: 检查`manifest.json`文件和图标是否完整，确保开启开发者模式。

**Q: AES加密失败？**  
A: 确认密钥是有效的Base64格式，长度正确（128位=24字符，256位=44字符）。

**Q: RSA加密文本过长？**
A: RSA有长度限制，2048位密钥最多加密245字节，建议使用AES加密大文本。

**Q: 签名验证失败？**
A: 确保签名时和验证时使用相同的算法，消息内容完全一致。

**Q: 界面显示异常？**
A: 尝试刷新扩展或重启浏览器，清除浏览器缓存。
